## Instrucciones del proyecto

- Esta instruccion aplica solo al proyecto `quennie-portfolio`.
- Al terminar una tarea que implique cambios en archivos de este repositorio git, hacer `git status`, ejecutar las verificaciones razonables, crear un commit con un mensaje descriptivo y hacer `git push`.
- No dejar cambios de codigo sin commit/push salvo que haya un bloqueo real, por ejemplo: no existe remoto configurado, faltan permisos/autenticacion, las pruebas fallan y no conviene subirlo, o hay cambios ajenos del usuario que no deben mezclarse. En ese caso, explicar el bloqueo de forma concreta.
